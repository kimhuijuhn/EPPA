# Dev Log — 2026-04-06

## What we did

### Project restructure
- Deleted the flat command-function files (`get_session_name`, `get_track_list`, etc.)
- Adopted an MVC architecture suited for a visually impaired user:
  - **Model**: `PlaybackController` — wraps PTSL, handles transport and solo
  - **View**: `Reader` — speaks feedback via macOS `say` command
  - **Controller**: `App` — wires MIDI input → pedal interpretation → PT actions
  - Supporting: `MidiInputHandler`, `PedalInterpreter`, `PedalZone`

### Pedal zone logic
- Expression pedal CC value (0–127) maps to zones: OFF / DEAD_LOW / SOLO / DEAD_HIGH / ON
- Dead bands provide hysteresis to prevent zone boundary toggling
- Zone thresholds: OFF=0–15, SOLO=30–97, ON=110–127

### Zone transition rules implemented
| Transition | Action |
|---|---|
| stop → solo | Solo selected track, start playing |
| play → solo | Solo selected track, keep playing |
| solo → play | Unsolo, keep playing |
| solo → stop | Unsolo, stop |

### Bugs fixed
- `SetTrackSoloState` JSON fields were wrong (`track_name`/`solo_state` → `track_names`/`enabled`)
- `GetTransportState` returns deprecated enum names (`TS_TransportPlaying` not `TState_TransportPlaying`) — fixed comparison to handle both
- SOLO→PLAY was stopping the transport — fixed by skipping the `IsPlaying()` check when we already know PT is playing
- RtMidi was compiled without CoreMIDI backend — added `-D__MACOSX_CORE__` to CMakeLists.txt
- OFF zone threshold was too tight (0–2); widened to 0–15 to match real pedal resting position

### Reader temporarily disabled
- `Reader::Speak` calls `say` synchronously on the MIDI callback thread, blocking zone transitions
- Stubbed out for now; will need a background thread + message queue before re-enabling
