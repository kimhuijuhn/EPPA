# EPPA

> 영어 버전은 [English](README.md)를 참조하세요.

## 프로젝트 개요

EPPA는 **시각 장애가 있는 음악가 및 오디오 엔지니어**를 위한 핸즈프리 재생 제어 시스템입니다. 익스프레션 페달의 연속적인 입력값을 명확한 재생 상태로 매핑하여, 화면이나 마우스, 키보드를 보지 않고도 DAW(Pro Tools)를 조작할 수 있도록 합니다.

## 기술 스택

- **C++17** — 핵심 애플리케이션 언어. 저지연 실시간 오디오/MIDI 처리와 PTSL SDK에 대한 네이티브 접근을 위해 선택했습니다.
- **CMake** — 빌드 시스템. 플랫폼 프레임워크 및 SDK 헤더와의 링크를 일관되게 관리합니다.
- **PTSL SDK (Pro Tools Scripting Library)** — Pro Tools의 트랜스포트(재생 / 솔로 / 정지)를 프로그래밍 방식으로 제어합니다. 페달 입력을 실제 DAW 동작으로 변환하는 핵심 레이어입니다.
- **RtMidi** — 크로스 플랫폼 MIDI 입출력 라이브러리. 익스프레션 페달의 CC(Continuous Controller) 메시지를 실시간으로 수신합니다.
- **CoreMIDI / CoreAudio / CoreFoundation (macOS)** — RtMidi와 PTSL이 동작하기 위한 네이티브 프레임워크. Apple Silicon에서 저지연 MIDI 입력에 필수적입니다.
- **macOS `say` / 보이스오버 레이어 (`Reader`)** — 상태 전환을 음성으로 알려주어 사용자가 시각적 단서 없이도 동작을 확인할 수 있게 합니다. 접근성이라는 목표상 매우 중요한 부분입니다.

## 주요 기능

### 페달 존(Zone)에 따른 재생–솔로–정지
페달의 연속적인 입력 범위를 `OFF`, `SOLO`, `ON`의 명확한 존으로 분할하고, 각 존 사이에 `DEAD_LOW` / `DEAD_HIGH` 히스테리시스 구간을 두어 경계에서의 지터를 억제합니다. 각 존은 Pro Tools 트랜스포트 동작에 매핑됩니다.

- **OFF** (페달 뗀 상태) → **정지(Stop)**
- **SOLO** (중간 위치) → 현재 트랙 **솔로(Solo)**
- **ON** (완전히 밟은 상태) → **재생(Play)**

데드 밴드를 통해 상태 전환이 디바운싱되므로, 경계 근처에서 페달이 흔들려도 재생이 빠르게 토글되지 않습니다.

## 트러블슈팅

### 페달 타입 호환성 (TS vs. TRS)
현재 **TS(모노)** 와 **TRS(스테레오)** 익스프레션 페달 간의 극성 / 배선 이슈를 다루고 있습니다. 제조사별로 와이퍼와 기준 핀을 팁(tip)과 링(ring)에 다르게 배정하기 때문에, 호스트가 받는 CC 값이 반전되거나 변하지 않는 경우가 발생합니다. **이는 진행 중인 조사 항목**으로, 여러 페달 모델에서의 동작을 측정하고 있으며 소프트웨어 측에서 극성 / 핀 매핑을 설정할 수 있는 옵션을 검토 중입니다.
